# Capstone Curriculum

## Capstone Summary

**Title**: MediNotes Pro — Full-Stack AI Healthcare SaaS

**Elevator Pitch**: Build a production-grade, streaming AI application with authentication, subscriptions, Docker containers, and dual cloud deployment — using a healthcare consultation assistant as the vehicle.

**Difficulty Level**: Intermediate to Advanced

**Estimated Time**: 40-60 hours (5 focused days at 8-12 hours each, or 4-6 weeks part-time)

**Prerequisites**:
- Working knowledge of JavaScript/TypeScript and Python
- Basic familiarity with React (components, hooks, state)
- Basic command line proficiency
- No prior cloud deployment experience required

---

## Learning Objectives

By completing this capstone, you will be able to:

1. Build a full-stack application with a React/TypeScript frontend and Python/FastAPI backend
2. Implement JWT-based authentication with JWKS verification
3. Integrate subscription billing into a SaaS product
4. Design and implement Server-Sent Events for real-time streaming
5. Engineer structured prompts for reliable LLM output
6. Containerize applications with multi-stage Docker builds
7. Deploy to both serverless (Vercel) and container (AWS App Runner) platforms
8. Configure cloud cost controls and monitoring

---

## Project Stages and Milestones

### Milestone 1: API Foundation (Day 1)

**Objective**: Deploy a working API endpoint to the public internet.

**Deliverables**:
- FastAPI application with a single GET endpoint
- Vercel deployment configuration
- OpenAI API integration returning dynamic content
- Environment variable management for API keys

**Acceptance Criteria**:
- [ ] `GET /` returns a response when accessed via browser
- [ ] Response content is dynamically generated by OpenAI
- [ ] API key is stored in Vercel environment variables, not in code
- [ ] Application is accessible via HTTPS on a public URL

**Skills Exercised**: Python basics, FastAPI routing, CLI deployment, environment variables

### Milestone 2: Full-Stack Application (Day 2)

**Objective**: Build a complete frontend-backend application with real-time streaming.

**Deliverables**:
- Next.js frontend with TypeScript and Tailwind CSS
- SSE streaming from FastAPI to browser
- Markdown rendering of streamed content
- Professional styling with dark mode support

**Acceptance Criteria**:
- [ ] Frontend loads and displays streamed AI content
- [ ] Markdown renders correctly (headings, lists, bold)
- [ ] Dark mode works via Tailwind `dark:` classes
- [ ] Both frontend and backend deploy together on Vercel

**Skills Exercised**: React hooks, TypeScript, SSE protocol, CSS frameworks, Markdown rendering

### Milestone 3: Authentication and Billing (Day 3)

**Objective**: Add user authentication, route protection, and subscription management.

**Deliverables**:
- Clerk authentication with multi-provider support
- Landing page with sign-in flow
- Protected product page with subscription gate
- Backend JWT verification via JWKS
- Pricing table for non-subscribers

**Acceptance Criteria**:
- [ ] Unauthenticated users see sign-in button on landing page
- [ ] Authenticated users can navigate to product page
- [ ] Non-subscribers see pricing table on product page
- [ ] API rejects requests without valid JWT (returns 401/403)
- [ ] JWT `sub` claim provides user identity on backend

**Skills Exercised**: OAuth/JWT concepts, authentication providers, authorization patterns, subscription SaaS model

### Milestone 4: Domain Application (Day 4)

**Objective**: Transform the application for healthcare use with structured input and output.

**Deliverables**:
- Consultation form with patient name, date picker, and notes textarea
- POST endpoint with Pydantic validation
- System prompt engineering for three-section output
- Healthcare-specific landing page and UI

**Acceptance Criteria**:
- [ ] Form collects patient name, date, and notes
- [ ] POST request sends structured JSON to backend
- [ ] Backend validates input with Pydantic model
- [ ] Output contains three distinct sections: summary, next steps, email
- [ ] Date picker formats dates in ISO format

**Skills Exercised**: Form handling, API design (POST), input validation, prompt engineering, domain-specific UI

### Milestone 5: Containerization and Cloud Deployment (Day 5)

**Objective**: Package the application as a Docker container and deploy to AWS.

**Deliverables**:
- Multi-stage Dockerfile (Node.js builder + Python runtime)
- Next.js static export configuration
- FastAPI server serving both API and static files
- AWS ECR repository with pushed image
- App Runner service running the container
- Health check endpoint
- Budget alerts configured

**Acceptance Criteria**:
- [ ] Docker image builds successfully on any platform
- [ ] Container runs locally and serves the full application
- [ ] Image is pushed to ECR
- [ ] App Runner service shows "Running" status
- [ ] Health check endpoint returns 200
- [ ] Application works end-to-end on AWS URL
- [ ] Budget alerts are configured at $1, $5, $10

**Skills Exercised**: Docker, multi-stage builds, static export, AWS services (ECR, App Runner, IAM, Budgets), cost management

## Challenge Tiers

### Beginner Challenges

1. Modify the system prompt to add a fourth output section (e.g., "Billing codes")
2. Add a "Generate New" button that clears the form and output
3. Change the landing page styling (colors, layout, copy)
4. Add a character count indicator to the notes textarea

### Intermediate Challenges

5. Add a specialty dropdown that modifies the system prompt per specialty
6. Implement a "Copy to Clipboard" button for each output section
7. Add a loading progress indicator that shows estimated completion
8. Implement client-side input validation (min/max note length)
9. Add a `/history` page stub that will eventually show past consultations
10. Write a pytest test suite for the Pydantic model and prompt construction

### Advanced Challenges

11. Add PostgreSQL database to persist consultation results (with user_id)
12. Implement per-user rate limiting middleware (token bucket algorithm)
13. Set up GitHub Actions CI/CD pipeline for automated AWS deployment
14. Add PDF export for the generated consultation summary
15. Implement React error boundaries with fallback UI
16. Add retry logic with exponential backoff for OpenAI API calls

### Stretch Goals

17. Add voice-to-text input using the Web Speech API
18. Implement multi-language patient email generation
19. Add WebSocket support as an alternative to SSE
20. Build an admin dashboard showing usage analytics per user

## Common Mistakes Learners Make

1. **Forgetting `--platform linux/amd64`** on Apple Silicon Macs — results in "exec format error" on AWS
2. **Using `@clerk/nextjs` v7** which removed `SignedIn`/`SignedOut` components — pin to v6.39.0
3. **Not adding environment variables to Vercel** before deploying — backend crashes with missing key errors
4. **Confusing the two API paths** — Vercel uses `/api` (from `api/index.py`), AWS uses `/api/consultation` (from `api/server.py`)
5. **Using native `EventSource` for POST** — it only supports GET. Must use `fetchEventSource`
6. **Forgetting to add `IAMFullAccess`** to the AWS user group — causes cryptic permission errors
7. **Signing up for AWS Free Tier sandbox** — App Runner is not available in the sandbox tier
8. **Not loading environment variables** before Docker build — the Clerk public key is baked in at build time
9. **Editing `node_modules`** instead of installed packages — changes are lost on `npm install`
10. **Committing `.env.local`** — secrets exposed in git history

## Interview-Style Questions

### System Design

1. "Walk me through the request lifecycle from form submission to rendered output."
2. "Why did you choose SSE over WebSockets? When would you switch?"
3. "How would you add a database to this system? What schema would you use?"
4. "Your OpenAI API starts returning errors 50% of the time. How do you handle this?"
5. "How would you implement rate limiting per user?"

### Authentication and Security

6. "Explain how JWT verification works without contacting Clerk on every request."
7. "What is JWKS and why does the backend fetch it?"
8. "If a JWT is stolen, what is the blast radius? How do you mitigate it?"
9. "What would you need to change for HIPAA compliance?"
10. "How do you prevent prompt injection through the notes field?"

### DevOps and Deployment

11. "Explain the multi-stage Docker build. Why two stages?"
12. "Why does the Apple Silicon Mac need `--platform linux/amd64`?"
13. "Compare your Vercel and AWS deployments. When would you choose each?"
14. "How would you set up CI/CD for this application?"
15. "Your AWS bill jumped to $50 this month. How do you investigate?"

### Frontend and API

16. "How does streaming work from the client's perspective? What is the event buffer?"
17. "Why can't you use native `EventSource` for this endpoint?"
18. "How does Clerk's `<Protect>` component work with subscription gating?"
19. "What happens if the user refreshes during streaming?"
20. "How would you add offline support or caching?"

### Prompt Engineering

21. "How do you ensure the LLM reliably produces three sections?"
22. "What happens if the doctor's notes are very short? Very long?"
23. "How would you handle LLM hallucinations in a medical context?"
24. "How would you A/B test different system prompts?"
25. "What is the cost per consultation at current token pricing?"
